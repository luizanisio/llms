# ⚡ REDUÇÃO DE USO DE VRAM — flash attention + liger-kernel
# ==============================================================================
# São COMPLEMENTARES: flash attention atua no bloco de atenção;
# liger-kernel atua no restante (RMSNorm, RoPE, Cross-Entropy).
#
# ------------------------------------------------------------------------------
# 🔥 flash attention — atenção O(n) VRAM vs O(n²) (nunca materializa matriz NxN)
# ------------------------------------------------------------------------------
#
# OPÇÃO 1 — SDPA nativo do PyTorch >= 2.0 (RECOMENDADA, sem instalação extra)
#   Ative com: flash_attention_2: true  no YAML de treinamento.
#   O código detecta torch.backends.cuda.flash_sdp automaticamente.
#   Economia de VRAM equivalente ao pacote pip para LoRA/QLoRA fine-tuning.
#   Requisito: PyTorch >= 2.0 com CUDA (Ampere+, inclusive RTX 3060 sm_86).
#
# OPÇÃO 2 — Pacote pip flash-attn (opcional, apenas para features avançadas)
#   Kernels customizados (Tri Dao) — marginalmente mais rápido em seq. longas.
#   Necessário apenas para: sliding window attention, paged attention, FA-3 no H100.
#   ⚠️  DEVE ser compilado do source contra o PyTorch instalado (ver Passo 4).
#   ❌ NÃO instalar via requirements.txt nem usar wheels pré-compilados
#      (geram erro "undefined symbol" por incompatibilidade de ABI).
#   Hardware: requer Ampere+ (sm_80+). T4/Turing: não suportado.
#   ABI rígida: upgrade do torch invalida o binário — recompilar após qualquer update.
#
# flash-attn   # <-- OPCIONAL; ver Passo 4 ao final
#
# ------------------------------------------------------------------------------
# 🦁 liger-kernel — Triton kernels para LLM training (LinkedIn)
# ------------------------------------------------------------------------------
# Patcheia camadas não-atenção com kernels Triton fundidos (RMSNorm, RoPE,
# SwiGLU, FusedLinearCrossEntropy). Instala via pip em segundos, sem compilação.
#
# Benefícios principais:
#   • FusedLinearCrossEntropy: NUNCA materializa o tensor de logits [B, T, V].
#     Em Gemma (V=256K) com seq=4K e batch=8: economia de ~16 GB em bfloat16.
#   • +20% throughput, -60% VRAM em multi-GPU (benchmark LinkedIn, LLaMA-3 8B).
#   • Habilita contextos 4–8x maiores na mesma GPU.
#
# Caveats importantes:
#   • ❌ device_map="auto": NÃO FUNCIONA (kernels assumem mesma GPU para entrada e pesos).
#   • ⚠️  FSDP + PEFT com modules_to_save: desabilitar fused_linear_cross_entropy=False.
#   • ⚠️  Tensor/Sequence Parallel: desabilitar patches que tocam linear sharded.
#   • Cobertura limitada por arquitetura — modelos novos dependem de PR no repo.
#
# 📊 QUANDO USAR:
#   ┌────────────────────────────────┬──────────────┬──────────────┐
#   │ Cenário                        │ flash-attn   │ liger-kernel │
#   ├────────────────────────────────┼──────────────┼──────────────┤
#   │ Single-GPU SFT/LoRA            │ ✅ SDPA/pip  │ ✅ Sim       │
#   │ Multi-GPU FSDP/DDP             │ ✅ Sim       │ ✅ Sim       │
#   │ Multi-GPU + LoRA + embeddings  │ ✅ Sim       │ ⚠️  Cuidado* │
#   │ DeepSpeed ZeRO-3               │ ✅ Sim       │ ✅ Sim       │
#   │ Tensor / Sequence Parallel     │ ⚠️  Verificar│ ❌ Desligar  │
#   │ device_map="auto"              │ ✅ Sim       │ ❌ Quebra    │
#   │ H100 / Hopper                  │ ✅ FA-3 FP8  │ ✅ Sim       │
#   │ RTX 3060 / Ampere consumer     │ ✅ SDPA/FA-2 │ ✅ Sim       │
#   │ T4 / Turing                    │ ❌ pip N/A   │ ✅ Sim       │
#   └────────────────────────────────┴──────────────┴──────────────┘
#   * Desabilitar fused_linear_cross_entropy quando lm_head sharded em FSDP + PEFT.
#

# 🆘 SOLUÇÃO DE PROBLEMAS
# ==============================================================================
#
# ❌ "CUDA_HOME environment variable is not set" (ao compilar flash-attn):
#   → nvcc não está no PATH. Verificar: which nvcc
#   → Exportar antes do pip: export CUDA_HOME=...; export PATH=$CUDA_HOME/bin:$PATH
#
# ❌ "undefined symbol: _ZN3c104cuda..." (ao importar flash-attn):
#   → flash-attn compilado contra CUDA diferente do PyTorch.
#   → Garantir CUDA_HOME casado com torch.version.cuda e recompilar (Passo 4).
#
# ❌ "No space left on device":
#   → Usar TMPDIR=/var/tmp antes do pip.
#
# ❌ Liger + FSDP: "Output 0 of ViewBackward0 is a view... modified inplace":
#   → Conflito FLCE + FSDP-sharded lm_head + PEFT modules_to_save.
#   → Desabilitar apenas o FLCE: fused_linear_cross_entropy=False no patcher.
#   → Alternativa: fsdp_use_orig_params=True ou migrar para DeepSpeed ZeRO-3.
#
# ❌ Liger + device_map="auto": ValueError em forward:
#   → Liger não suporta model parallel. Migrar para FSDP/DeepSpeed ou desativar Liger.
#
# ❌ FA-2 lento no H100 (<40% utilização):
#   → FA-2 não usa TMA/WGMMA do Hopper. Compilar branch hopper do flash-attn (FA-3).
#
# ⚠️  pip atualizou o torch e quebrou dependências:
#   → Reverter: pip install "torch==2.11.0" torchvision --index-url https://download.pytorch.org/whl/cu128
#   → Realinhar: pip install "setuptools==80.0.0" "numpy<2.4.0" "cuda-python<13" "cuda-bindings<13" "fsspec<=2026.2.0"
#   → Qualquer troca de versão do torch invalida o flash-attn compilado — recompilar.
